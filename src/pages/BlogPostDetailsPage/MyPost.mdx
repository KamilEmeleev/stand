## Вступление

Работая с библиотекой компонентов, мы часто сталкиваемся с проблемой, когда стили,
переданные через `className`, не перебивают дефолтные стили компонента. Это может быть вызвано тем,
что специфичность ваших стилей недостаточно высока для того, чтобы переопределить существующие правила.
В таком случае одно из решений — повысить специфичность.

Пример для компонента кнопки:
```css
.Button_variant_contained.Button_color_primary {
  /** Стили определённые библиотекой для кнопки с заданным цветом и вариантом */
}

.yourClassName {
  /* Здесь ваши стили не перебивают дефолтные */
}
```

**Простой и легальный лайфхак для решения этой проблемы — увеличить кратность класса в CSS-селекторе,
без изменения HTML-тега компонента**.

## Увеличение кратности СSS-селектора

**HTML-тег остается неизменным**:

```tsx
() => (
  <Button className="yourClassName">
    Содержимое компонента
  </Button>
)
```

**В CSS увеличиваем кратность своего класса вдвое**:

```css
.yourClassName.yourClassName {
  /* Ваши стили */
}
```

Дублируя класс в селекторе, вы увеличиваете его специфичность вдвое, что позволяет вашим стилям перебить дефолтные
без определения родительского контейнера или использования конструкции `!important`.

## Заключение

Увеличение кратности класса — это простой и эффективный способ, который позволяет обойти проблему с дефолтными стилями.
Однако важно использовать этот метод осторожно, чтобы избежать чрезмерного усложнения CSS-кода и поддержания
гибкости в будущем.
