# Внести вклад

**Özen-UI** — библиотека для реализации веб-интерфейсов разработанная согласно
правилам и принципам дизайн-системы **Bereke Bank Design**.

**В развитии библиотеки компонентов можно участвовать, мы рады вашим предложениям и доработкам.**

Основные ресурсы:

- [Figma](https://www.figma.com/files/1246048263126703364?fuid=1260967358201830178)
- [Github](https://github.com/bereke-bank/ozen-ui)
- [Telegram-канал](https://t.me/+SIGIw42UevI5ZWEy)

## Быстрый старт

- Склонировать репозиторий и перейти в созданную директорию
- Запустить в терминале команду `pnpm dev`

Документация будет доступна на http://localhost:6006. В ней ведётся вся разработка.
Для удобства можно сразу перейти на страницу разрабатываемого компонента.

## Технологический стек

Библиотека компонентов дизайн-системы существуют в React-реализации.

Özen-UI может использоваться для создания проектов любого уровня сложности и масштабов:
от простых презентационных страниц-лендингов до сложных админ-панелей и мобильных интерфейсов.

Основной стек:

- [React](https://react.dev/)
- [Typescript](https://www.typescriptlang.org/)
- [PostCSS](https://postcss.org/)

Мы поощряем и вознаграждаем усилия в развитие дизайн-системы со стороны

### Внести предложение

В большинстве случаев формирование issue является первым шагом к внесению вклада
в пользовательский интерфейс Özen-UI. Проверьте существующие issues и убедитесь,
что ваша проблема еще не описана. Затем используйте один из шаблонов, чтобы **задать вопрос**,
**сообщить о проблеме** или **предложить улучшение**.

Подробнее об этом [здесь](?path=/docs/introduction-feedback--docs).

### Запрос на внесение изменений

Ниже перечисленны основные требования которые необходимо учесть при запросе
на внесения изменений в библиотеку (pull request):

- Убедитесь, что ваш PR не дублирует другой
- Запрос направлен в правильную ветку
- Проверены все изменения, ничего лишнего в PR не попало
- Прилинкованы затронутые issue и связанные PR
- Коммиты проименованы в соответствии с [правилами](https://ozen-ui.netlify.app/?path=/docs/for-contributors-other-commit-naming-convention--docs)

## Ревью кода

После внесения изменений в библиотеку, разработчик предоставляет по средствам PR свой код для рассмотрения коллегам.
Они анализируют структуру, логику, соответствие стандартам написания кода библиотеки и оценивают качество.

### Правила ревью
- Допускаются PR, получившие не менее 2 аппрувов
- Распространяем новые знания
- При разногласиях созваниваемся и обсуждаем вопрос голосом

### Стоит помнить
- Ревью кода — одна из самых сложных операций в программировании
- Ревью может занимать много времени, это нормально
- Ревью — это процесс общения
- Читать чужой код всегда сложнее, чем писать свой

## Чеклист для компонента

- Дизайн: компонент разработан согласно дизайн-паттернам
- JS: нет варнингов и ошибок в консоли браузера
- Тесты: новый функционал и исправленные баги покрыты тестами
- Документация: отражены все изменения в API компонентов и описаны важные особенности реализации или использования
- Сторибук: для компонентов написаны или обновлены stories
- Верстка: используются CSS-переменные из токенов дизайн-системы
- Верстка: компонент учитывает адаптивную верстку
- Accessibility: компонент отвечает требованиям доступности

## Архитектура проекта

Проект библиотеки является **монорепозиторием** и включает в себя следующие пакеты:

- **@ozen-ui/kit** — библиотека React-компонентов, для создания веб-интерфейсов
- **@ozen-ui/icons** — коллекция иконок
- **@ozen-ui/fonts** — набор базовых шрифтов
- **@ozen-ui/logger** — утилита для вывода системного лога на стороне клиента

Проектирование и управление монорепозиторием осуществляется по средствам
[pnpm workspace](https://pnpm.io/workspaces) и [turborepo](https://turbo.build/repo/docs).

## Основные скрипты

- `pnpm dev` — установка зависимостей и запуск Storybook в режиме разработки
- `pnpm test` — запуск тестов для всех пакетов
- `pnpm lint` — запуск линтинга и форматирования кода для всех пакетов
- `pnpm build` — запуск сборки дистрибутивов для всех пакетов
- `pnpm release` — публикация релиза для всех пакетов
- `pnpm build-storybook` — сборка статики стенда storybook (./storybook-static)

Для более детальной информации по доступным и полезным скриптам обратитесь к файлам `package.json`.
